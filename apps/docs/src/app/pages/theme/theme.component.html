Theme

Create custom brand themes for your app without modifying tailng. 

tailng uses a token-based theming system built on CSS variables.

You apply:
- exactly one MODE (mode-light / mode-dark / ...)
- exactly one THEME (theme-default / theme-slate / ...)

App developers can add their own brand theme by defining the same semantic tokens (CSS variables) in their app â€” no fork required.
Important rule:
Use token-based utilities in templates (bg-bg, text-fg, etc.).

Avoid hard-coded Tailwind colors like bg-slate-* or text-slate-* because they break mode switching.

Apply mode + theme

Install theme package

Install @tailng-ui/theme if not already installed.

Where themes live

tailng provides tokens in:
@tailng-ui/theme/tokens/index.css

App adds brand themes by creating:
src/theme/theme-mybrand.css

Then importing it AFTER tailng tokens in:
src/styles.css

Step 1: Create a theme file in your app

Step 1) Create a new theme file in your app

Create:
src/theme/theme-mybrand.css

Naming convention:
theme-<name>  e.g. theme-mybrand, theme-acme, theme-foo

Step 2: Import the theme globally

Step 2) Import it globally AFTER tailng tokens

Order matters:
1) tailng tokens (defaults + system)
2) your app theme (override brand tokens)

Step 3: Activate your theme

Step 3) Activate your theme with one mode + one theme

You must apply:
- one mode-* class
- one theme-* class

Step 4: Use token-based utilities

Step 4) Use token-based utilities in templates

Correct: 

- bg-bg / text-fg
- components that use tokens (tailng)

Incorrect: 

- bg-slate-* / text-slate-* / text-white (hard-coded)

Optional: Multiple brand themes

Optional) Multiple brand themes

Create multiple files:
- src/theme/theme-acme.css
- src/theme/theme-enterprise.css

Switch by changing the class:
theme-acme / theme-enterprise
No rebuild required.

Checklist

Checklist

Troubleshooting

Troubleshooting

If the theme doesn't apply:
1) Confirm your theme file is imported AFTER tailng tokens in styles.css
2) Confirm your body/html has both classes:
   mode-* and theme-*
3) Confirm you didn't accidentally use hard-coded colors in templates
4) Ensure variable names match exactly:
   --primary-color, --danger-color, etc.
