Styling

The tng-option-list component uses slot-based micro styling via the slot input. The slot object accepts keys: container, option, optionActive, optionInactive, empty, and overlayPanel. Each slot targets a specific element in the component structure.

container slot

The container slot applies classes to the listbox <div> that wraps all options. Use this for padding, max-height, overflow behavior, and other container-level styling.

Example
Tip: Default classes include py-1 overflow-auto max-h-60. Adjust max-height and overflow based on your design needs.

option slot

The option slot applies base classes to all option <div> elements. Use this for padding, text styling, cursor, and other base option appearance.

Example
Note: Default classes include px-3 py-2 text-sm cursor-pointer select-none. These are merged with active/inactive state classes.

optionActive / optionInactive slots

The optionActive and optionInactive slots apply state-specific classes to options based on whether they are currently focused/active. Use these for background colors, text colors, and other state-dependent styling.

Example
Tip: Default active classes: bg-primary text-on-primary. Default inactive classes: bg-bg text-fg. Customize these to match your design system.

empty slot

The empty slot applies classes to the empty state message when no items are available. Use this for text styling, padding, and visual appearance of the "no results" message.

Example
Note: Default classes include px-3 py-2 text-sm text-disable. The empty text can be customized via the emptyText input.

overlayPanel slot

The overlayPanel slot applies classes to the tng-overlay-panel component when modal=true. This controls the panel wrapper appearance (padding, borders, etc.) around the option list in modal mode.

Example
Note: This slot only applies when modal=true. Default value is p-0 (no padding).

Best practices

  • • Use container slot for listbox container styling (max-height, overflow, padding).
  • • Use option slot for base option styling, and optionActive/optionInactive for state-specific styling.
  • • Ensure sufficient contrast between active and inactive states for accessibility.
