Styling

The tng-text-input component provides four klass hooks for customizing different parts of the input: rootKlass, inputKlass, prefixKlass, and suffixKlass. Each hook targets a specific element in the component structure.

rootKlass

The rootKlass input applies classes to the root wrapper <div> that contains the entire input structure. Use this for styling the container: borders, background, rounded corners, shadows, and focus ring styles.

Example
Tip: The default root classes include focus ring styling. When overriding rootKlass, you may want to include your own focus states for consistency.

inputKlass

The inputKlass input applies classes directly to the <input> element. Use this for text styling, padding adjustments, placeholder styling, and other input-specific customizations.

Example
Note: The default input classes include px-3for horizontal padding. Adjust this if you need different spacing.

prefixKlass

The prefixKlass input applies classes to the wrapper <span> that contains projected prefix content. Use this for styling the prefix area: background colors, borders, or other visual treatments.

suffixKlass

The suffixKlass input applies classes to the wrapper <span> that contains projected suffix content. Use this for styling the suffix area: background colors, borders, or other visual treatments.

Best Practices

  • • Use rootKlass for container-level styling (borders, shadows, focus rings).
  • • Use inputKlass for text and input-specific styling.
  • • Apply spacing classes (e.g., ml-3, mr-3) to projected elements, not the klass wrappers.
  • • When overriding rootKlass, include focus states for accessibility.
  • • The component merges your klass inputs with default classes, so you only need to override what you want to change.
