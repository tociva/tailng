Styling

The tng-text-input component uses slot-based micro styling via the slot input. The slot object accepts keys: frame, input, prefix, and suffix. Each slot targets a specific element in the component structure.

frame slot

The frame slot applies classes to the root wrapper <div> that contains the entire input structure. Use this for styling the container: borders, background, rounded corners, shadows, and focus ring styles.

Example
Tip: The default root classes include focus ring styling. When overriding frame slot, you may want to include your own focus states for consistency.

input slot

The input slot applies classes directly to the <input> element. Use this for text styling, padding adjustments, placeholder styling, and other input-specific customizations.

Example
Note: The default input classes include px-3 for horizontal padding. Adjust this if you need different spacing.

prefix slot

The prefix slot applies classes to the wrapper <span> that contains projected prefix content. Use this for styling the prefix area: background colors, borders, or other visual treatments.

Best Practices

  • • Use frame slot for container-level styling (borders, shadows, focus rings).
  • • Use input slot for text and input-specific styling.
  • • Apply spacing classes (e.g., ml-3, mr-3) to projected elements, not the slot wrappers.
  • • When overriding frame slot, include focus states for accessibility.
  • • The component merges your slot values with default classes, so you only need to override what you want to change.
