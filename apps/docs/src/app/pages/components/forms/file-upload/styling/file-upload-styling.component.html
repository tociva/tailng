Styling

The tng-file-upload component uses slot-based micro styling via the slot input. The slot object accepts keys: container, dropzone, dropzoneActive, dropzoneDisabled, header, title, subtitle, hint, clearButton, fileList, fileItem, fileName, and fileSize. Each slot targets a specific element in the component structure.

container slot

The root wrapper element. Default: w-full.

Example

dropzone slot

Applies to the drag-and-drop area (border, padding, focus ring).

Example

dropzoneActive slot

Applied when files are being dragged over the dropzone. Default includes border-primary ring styles.

Example

Drag files over the dropzone to see the active state

dropzoneDisabled slot

Applied when the component is disabled. Default includes opacity-60 pointer-events-none.

Example

header slot

The container for title, subtitle, and clear button. Default: flex items-start justify-between gap-4.

Example

title & subtitle slots

Style the dropzone title and subtitle text.

Example

hint slot

The hint text showing accepted file types. Default: text-xs text-disable.

Example

clearButton slot

The clear button that appears when files are selected. Default: text-xs text-danger hover:text-danger-hover.

Example

Select files to see the clear button

fileList slot

The container for the list of uploaded files. Default: mt-2 space-y-1.

Example

Upload files to see the file list

fileItem slot

Each individual file row in the list. Default includes flex, border, padding, and text styles.

Example

Upload files to see styled file items

fileName slot

The file name text within each file item. Default: truncate.

Example

Upload files to see styled file names

fileSize slot

The file size text within each file item. Default: ml-3 text-xs text-disable.

Example

Upload files to see styled file sizes
