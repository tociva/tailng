Styling

The tng-autocomplete component uses slot-based micro styling via the slot input. The slot object accepts keys: container, inputWrapper, input, selectedTpl, and overlayPanel. Each slot targets a specific element in the component structure.

container slot

The container slot applies classes to the root wrapper <div> that contains the entire autocomplete structure. Use this for styling the container: positioning, layout, and spacing.

Example
Tip: The default container class includes relative for positioning the overlay.

inputWrapper slot

The inputWrapper slot applies classes to the wrapper <div> that contains the input element. This wrapper handles disabled state styling (opacity and pointer-events). Use this slot for background colors, borders, padding, and other wrapper-level styling that should apply around the input.

Example
Note: The wrapper automatically applies opacity-60 and pointer-events-none when disabled. Slot classes are merged with these conditional classes.

input slot

The input slot applies classes directly to the <input> element. Use this for borders, width, rounded corners, focus styling, padding, and other input-specific customizations.

Example
Tip: Default classes include focus ring; when overriding, consider keeping focus states for accessibility.

selectedTpl slot

The selectedTpl slot applies classes to the overlay <div> that displays the selected value template (when using #inputTpl). This overlay appears inside the input when a value is selected and the dropdown is closed. Use this for background colors, borders, rounded corners, and other visual styling of the selected value display.

Example
({{ item.code }}) {{ item.name }}
Note: The selectedTpl slot only applies when using the #inputTpl template. Default classes include positioning (absolute, inset-y-0) and layout (flex items-center).

overlayPanel slot

The overlayPanel slot applies classes to the tng-overlay-panel component that wraps the option list. This is the dropdown panel that appears when the autocomplete is open. Use this for customizing the panel's appearance: borders, background colors, shadows, rounded corners, max-height, and overflow behavior.

Example
Note: The overlay panel slot is passed to the underlying tng-overlay-panel component. Default classes include border, shadow, rounded corners, max-height, and overflow handling.

Best practices

  • • Use input slot for input appearance and overlayPanel slot for dropdown panel styling.
  • • Keep or replicate focus styles for accessibility.
