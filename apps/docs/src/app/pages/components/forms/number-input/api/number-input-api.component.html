API reference for number input

Inputs

This table lists all inputs for tng-number-input: identity, state, numeric constraints, browser hints, and theming. Value type is number | null (empty yields null).

Identity & Basics

Standard input attributes for id, name, and placeholder.

State

Properties related to input state.

Numeric constraints

min and max are used for validation and optional clamping on blur. step affects the browser stepper and validation.

Browser Hints

Hints for mobile keypad and autocomplete.

Styling

Properties for customizing component appearance.

Slot-based micro styling

Use the slot input to customize styling for different parts of the component. The slot object accepts keys: frame, input, prefix, and suffix.

  • — shell wrapper
    — input element
  • — prefix wrapper
    — suffix wrapper
Example:
Tip: Slot values can be strings or arrays of strings. Prefer frame for height, radius, borders, and focus-ring changes to keep behavior consistent.

Content projection

  • — prefix content (icons, labels)
  • — suffix content (icons, buttons)
Spacing is controlled by the projected elements themselves (e.g. ml-3 / mr-3).

Forms integration

Implements ControlValueAccessor and works with both reactive and template-driven forms. Value is number | null; empty input or invalid number yields null.

  • Supports formControlName and ngModel
  • Values are clamped to min/max on blur when set
