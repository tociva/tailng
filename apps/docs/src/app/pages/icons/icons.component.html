Icons

Use tng-icon (powered by @ng-icons/core) to render scalable SVG icons in tailng apps. 

tailng icons are built on top of @ng-icons/core.

- You install @tailng-ui/icons + @ng-icons/core (peer dependency)
- You register one or more icon packs in your app
- Then you render icons via <tng-icon name="...">

Why do I need to install @ng-icons/core?

@tailng-ui/icons wraps ng-icons to keep bundles small and avoid shipping duplicated icon engines.
That's why @ng-icons/core is a peer dependency (you install it explicitly).

2) Register icons

Before you can use an icon name (like "bootstrapCopy"), you must register that icon in the ng-icons registry.

You can register:
- individual icons (recommended)
- or entire packs (not recommended for bundle size)

Best practices

Best practices

- Register only the icons you use (better tree-shaking and smaller bundles)
- Keep a single icons provider file in your app (easy maintenance)
- Use decorative icons by default; set ariaLabel only when the icon conveys meaning

Troubleshooting

Troubleshooting

1) Icon doesn't render:
- Confirm you installed @ng-icons/core
- Confirm the icon is registered via provideIcons(...)
- Confirm the name matches the registered key (case-sensitive)

2) Console says "No icon found for name ...":
- You are using a name that was not registered.

3) Bundle is too big:
- Avoid importing/registering full packs
- Register only the icons you use
